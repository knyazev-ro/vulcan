Vulcan - это ORM с полноценным Query билдером, заточенный под быструю разработку. Идея этой библиотеки была вдохновлена Eloquent ORM из потрясающего фреймворка Laravel. Vulcan старается быть максимально приближен к Elouqent в плане Query Builder, но держится ровно противоположного подхода в принципе работы. Вместо Active Record здесь реализован Data Mapper, а отношения реализованы, как поля структуры, а не методы класса.

Философия Vulcan исключает использование громоздких глобальных сущностей. Данные описываются контекстуально: там, где они нужны — в репозитории, контроллере или бизнес-логике. Относитесь к моделям в Vulcan, как к Data Transfer Object (DTO), нежели как к полноценным моделям. Хоть эти модели и позволяют подгружать разного рода отношения, но в сущности они все так же выполняют роль лишь переносчика информации.

В примере ниже представлена реализация такой модели. 
```go
	type ReportCity struct {
		_        string `type:"metadata" table:"report_cities" pk:"id"`
		Id       int64  `type:"column" col:"id"`
		ReportId int64  `type:"column" col:"report_id"`
		City     string `type:"column" col:"city"`
		Order    int64  `type:"column" col:"order"`
	}
```
Модели Vulcan НЕ являются абсолютным представлением данных из таблицы! Они являются лишь либо частичным, либо полным представлением. Vulcan избавляет вас от проблемы избыточного получения данных (over-fetching). База данных отдает ровно столько, сколько ожидает ваша структура.

То есть в рамках проекта при обращении к таблице `report_cities` вы точно так же можете достать только поля `id` и `city`:
```go
	type ReportCityOnlyName struct {
		_        string `type:"metadata" table:"report_cities" pk:"id"`
		Id       int64  `type:"column" col:"id"`
		City     string `type:"column" col:"city"`
	}
```

Структура моделей определяет запрос к базе данных. То есть, указав в структуре только поля `city` и `id`, на выходе только эти поля будут получены. Это упрощает обращения к БД и создает некоторый контракт. Вы знаете как будут выглядеть данные на выходе еще до самого запроса, просто описав структуру.

Vulcan был заточен под чистую архитектуру, поэтому он будет идеальным дополнением для репозиториев.
